üõ†Ô∏è Technical Implementation

I created a comprehensive reflection management system with three key components: First, the Python backend (save_entry.py) uses json module to load existing data, append new reflections with timestamps and categories, and save updated data. Second, the frontend (reflections.html) fetches the JSON file asynchronously and parses it for DOM rendering. Third, the JavaScript layer implements interactive features including category filtering, statistics calculation, and CSV export. The system gracefully handles edge cases like missing files, invalid JSON, and network errors.

üí≠ Answers to Reflection Questions

Q1: How is storing data in a JSON file different from using browser storage?

JSON file storage persists on the server/backend side and can be modified by server-side scripts like Python. Browser storage (localStorage, sessionStorage) is client-side only and limited to 5-10MB per domain. JSON files allow structured, portable data that can be shared, backed up, and manipulated programmatically. Browser storage is ideal for user preferences, while JSON files enable collaborative data management.

Q2: How did you use Python to create or update your JSON file?

I created save_entry.py that imports the json module and os module. The script loads existing reflections.json, appends new entries with timestamps and metadata, and writes the updated data back. Each reflection object contains date (ISO format), text, category, and unique ID. The script provides an interactive menu for users to add new reflections or view existing ones, demonstrating CRUD operations on JSON files.

Q3: What does your PWA show locally vs on GitHub? Are they the same? Why or why not?

Locally, the PWA displays all features including dynamically loaded JSON reflections with real data. On GitHub Pages, users will see the HTML structure and styling, but the JSON fetch may fail due to CORS policies when loading backend/reflections.json from GitHub. This limitation demonstrates why full-stack applications often require hosted backends. However, the GitHub version still displays the page structure and is fully functional for learning demonstration purposes. Static JSON files work, but dynamic Python updates won't execute in GitHub's static hosting environment.

Q4: What extra feature did you add to your PWA using the JSON file, and why?

I implemented a CSV export feature that allows users to download their reflections as a spreadsheet. This feature adds significant value because: (1) it enables data portability and archival, (2) users can import data into analysis tools or documentation platforms, (3) it demonstrates advanced JavaScript concepts like blob creation and download triggering. I also built a comprehensive statistics dashboard that calculates metrics from the JSON data in real-time, providing users with insights about their reflection patterns and writing habits. Both features enhance the PWA's utility beyond simple display functionality.

üöÄ Challenges & Solutions

  • CORS Issues: Initially faced cross-origin errors when testing locally. Solved by running a local Python server (python -m http.server) instead of using file:// protocol
  • JSON Parsing: Implemented try-catch error handling for invalid JSON, ensuring graceful degradation when files are corrupted or missing
  • Date Formatting: Created utility functions to parse ISO format dates from Python and render them in user-friendly format
  • Data Synchronization: Managed browser caching by implementing refresh button to reload JSON data without page refresh
